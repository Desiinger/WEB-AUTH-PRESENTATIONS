Variable Scope

JavaScript has two scopes – global and local. Any variable declared outside of a function belongs to the global scope, and is therefore accessible from anywhere in your code. Each function has its own scope, and any variable declared within that function is only accessible from that function and any nested functions

The DOM in general

The Document Object Model is a crossplatform and language neutral interface that will allow programs and scripts to dynamically access and update the content, structure and style of HTML, XHTML and XML documents.

Why?

Developers wished to have better integration with HTML and JavaScript. Intention was to add interaction in various parts of the HTML code.

Developers used add-hoc ways to add interaction and dynamic behaviour.

For those purposes W3C developed the DOM specification.

Specifications

Each web document can be treated as a tree structured collection of objects, called the DOM tree

Objects can be accessed and manipulated using methods

The public interface of a DOM is specified by its application programming interface (API).

What is the DOM

The HTML Document Object Model (DOM):

An API that defines a standard way of accessing and manipulating an HTML document.

It presents a HTML document as a tree-structure (a node tree) with elements, attributes and text.

allows programmers to build documents, navigate their structure, and add, modify, or delete elements and content.

Almost anything found in an HTML document can be accessed, changed, deleted, or added using the DOM (with a few exceptions).

The Dom Wars

There is more than one DOM

There are several DOMs in existence:

The Legacy DOM: introduced in early versions of JavaScript language.

It is well supported by all browsers, but allows access only to certain key portions of documents, such as forms, form elements, and images.

The IE4 DOM: This document object model was introduced in Version 4 of Microsoft's Internet Explorer browser.

IE 5 and later versions include support for most basic W3C DOM features.

The W3C DOM: allows access and modification of all document content and is standardised by the World Wide Web Consortium (W3C). This model is supported by almost all the modern browsers.

DOM4 (Nodes and objects)

Core DOM currently at version 4 and uses Object Oriented concepts.

Objects (encapsulation of data and methods).

Methods (to access or change an object’s state).

Interfaces (declaration of a set of methods).

Each node in the tree is an object.

Root node is the document.

Targeting elements in the DOM

Remember, an id can be defined in each HTML element start tag.

A <div> with an id can be used as a node for a dynamic subtree

A <span> with an id can be used as a dynamic inline element

Accessing nodes by ID is generally the preferred way to access DOM elements with JavaScript

Accessing nodes by tag name

Access to elements by their tag name. Typically, we have many elements with the same tag, e.g. <li>, <a> etc.

document.getElementsByTagName("tag")
Returns the collection of elements of the specified tag (type)

The collection has a length attribute

An item in the collection can be reached by its index, e.g. var x=document.getElementsByTagName("p")[0];

Traversing the DOM tree (Moving up and down)

Traversal methods include:

childNodes: returns a collection of child nodes
Has a length attribute
Items can be reached by its index E.g. var body=html.childNodes[5];

parentNode: reference to parent node

firstChild: first child in list of child nodes

lastChild: last child in list of child nodes

previousSibling: reference to node preceding this node (if none, returns null)

nextSibling: reference to node after this node (if none, returns null)

hasChildNodes(): true if node has child nodes
