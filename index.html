How the 'interweb' works!

Computers connected to the web are called clients and servers. A simplified diagram of how they interact might look like this:

Clients

Clients are the typical web user's internet-connected devices (for example, your computer connected to your Wi-Fi, or your phone connected to your mobile network) and web-accessing software available on those devices (usually a web browser like Firefox or Chrome).

Servers (Azure)

Servers are computers that store webpages, sites, or apps. When a client device wants to access a webpage, a copy of the webpage is downloaded from the server onto the client machine. The server sends the website's files to the browser as a series of small chunks called data packets via a http request. This is displayed in the user's web browser.

HTTP Architecture

Hypertext Transfer Protocol
(The basis upon which the web is built)

HTTP was originally proposed by Tim Berners-Lee, the pioneer of the World Wide Web who designed the application protocol with simplicity in mind, to perform high-level data communication functions between Web-servers and clients

HTTP 1.0 (slooooow)

Is a protocol for request response in a client-server model

HTTP was designed for requesting documents

HTTP 1.0 was fine for a single document request from the server (As the web grew, quicker response times were needed)

In HTTP 1.0 a separate connection was made for every request to the serve

HTTP 1.1 (what you are prob using now)

HTTP 1.1 added reusuable connections Browsers could initialize a connection to a web server to retrieve the html page, then reuse the same connection to retrieve resources, images and so on.

Reduced latency, by reducing number of http connections required

HTTP2 (limited availability)

Parallel multiplexed requests and response do not block each other.

A single TCP connection is used to ensure effective network resource utilization despite transmitting multiple data stream

Reduced latency, faster web performance, better search engine rankings.

The History of HTML

When you look at a web page, you might wonder how it is made. To display a web page, your Internet browser reads and interprets a special set of instructions. These instructions are written primarily in two languages: HTML and CSS.

What is HTML?

Hyper Text Markup Language.

What is Hyper Text?

What is a Markup Language?

Hyper Text

Hypertext is text that links to other information. By clicking on a link in a hypertext document, a user can quickly jump to different content. Though hypertext is usually associated with Web pages, the technology has been around since the 1960s

Markup

HTML is a type of markup language. It encapsulates, or “marks up” data within HTML tags, which define the data and describe its purpose on the webpage

*Not a Programming Language

Programming languages have functional purposes. HTML, as a markup language doesn’t really “do” anything in the sense that a programming language does. HTML contains no programming logic

Web pages display content and have an underlying structure defined by HTML. The content can include text, images, links, or even audio and video.

Most HTML elements are written using opening <element>and closing </element>

Tags tell the browser what type of content to display. The tags are written in a specific order to define the structure of the page.

Example of three common HTML tags.

<h1> specifies heading text, good for page titles

<p> specifies paragraph text, good for body text

<img> displays an image specified by a src (source file)

Note:it is best practice to write HTML tags in lower case when coding your own pages.

What is CSS?

What is CSS?

With HTML you've given a basic structure to your web page, but there's not much customization to the style or layout. Eventually you want to add colors, format text and images, and reposition elements on your page.

What is CSS?

CSS is a language that works with HTML to apply this style and formatting to your web page. Use CSS to find one or more HTML elements, and then provide additional instructions for what to do with that element (such as change its color or center it on the page).

W3C

The World Wide Web Consortium (W3C) defines the web standards, or specifications, for these languages. It is important to understand web standards to ensure that the different web browsers display your pages accurately.

Editors

Editors like Sublime and Dreamweaver include features that allow you to build web pages based on the latest web standards. However specifications can ambigious and each browser (Chrome, FF, Edge, Opera) interpret them differently

Do not assume everyone is up-to-date!

Setting up your site Cache

Local Folder: This is your working directory—usually a folder on your hard drive. Editors (Dreamweaver) refer to this folder as your local site root

This repesents the server

This is where optimised html, css, js, images, videos etc are kept

This is NOT where development files are kept, i.e photoshop files etc

Why Define a local folder?

A common beginner mistake is to start working on web pages in without defining a site. This results in links to assets, such as images and style sheets, pointing to a location on your local computer (cdrive\desktop\wayne\folder) Everything works fine until you come to upload the site to your remote server. Then you discover all your links are broken and need to be fixed.

We reference all assets via relative links (relative to the current page)

linking to an image then becomes "/assets/imgname.jpg"

or linking to a style sheets becomes "/css/style.css"

(Makes it much easier to update)
